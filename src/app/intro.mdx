## **Next.js 简介**

回首前端技术的发展历程，前端开发这十年来，主流的开发方式从之前的 `jsp/php/asp` 等服务端渲染方案，逐渐演变成纯客户端渲染的技术方案。在这期间，Angular、React、Vue 粉墨登场，主宰了前端开发的一切。

**但是事实上，服务端渲染一直是一个巨大的刚需。** 他虽然没有被更普遍的提及，但是在大多数团队中，他从来没有消失。只是很多时候他**并不被初级前端工程师所关注**，很多入行不久的前端工程师对服务端渲染方案并不是太了解。

以至于有的人还认为，服务端渲染只停留在了一个提升首屏渲染速度的八股文概念这个层面。或者有的团队，干脆就将这样的需求，交给后端团队来做。比如用 python web 的框架。


![](https://files.mdnice.com/user/3925/1c776a5a-50c3-4041-980c-9c392f4a6b17.png)

有的后端语言在服务端渲染上的方案层出不穷，可能比前端方案发展得更好。**前后端分离的趋势，在一定程度上降低了开发者的准入门槛**。这种环境下滋生出来的前端工程师，天生就注定了在技术方案的积累上是残缺的，不懂服务端，不擅长服务端渲染是许多前端工程师进阶的主要壁垒和障碍。

因此，系统化掌握一门服务端渲染开发框架，是作为一名高阶前端工程师的必备技术能力。而 Next.js，是当前市面上，最热门、最完善、理念最先进的服务端渲染框架。
